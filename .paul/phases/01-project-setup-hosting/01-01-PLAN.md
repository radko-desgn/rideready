<objective>
## Goal
Prepare the existing RideReady prototype for production deployment: add deployment config, push to GitHub, deploy to Vercel, and wire in the Supabase SDK so it's ready for auth in Phase 2.

## Purpose
The app currently runs only locally. This phase makes it publicly accessible via a free Vercel URL, version-controlled on GitHub, and connected to Supabase — the foundation every subsequent phase depends on.

## Output
- Live URL on Vercel (e.g. rideready.vercel.app)
- GitHub repo with the project pushed
- Supabase project created with URL + anon key stored in index.html
- Supabase JS SDK loaded via CDN in index.html
</objective>

<task type="checkpoint:human-action" gate="blocking">
  <what-to-do>Create a Supabase project and get your credentials</what-to-do>
  <steps>
    1. Go to supabase.com → Sign up / Log in (free)
    2. Click "New project"
    3. Name: rideready
    4. Database password: choose a strong password and save it somewhere
    5. Region: pick the one closest to you
    6. Click "Create new project" — takes ~1 minute
    7. Once ready, go to: Project Settings → API

       - Project URL (looks like: https://xxxx.supabase.co)

  </steps>

<task type="auto">
  <name>Task 2: Add Supabase SDK and initialise client in index.html</name>
  <files>index.html</files>
  <action>
    In index.html, just before the closing </head> tag, add the Supabase CDN script:
    ```html
    <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
    ```

    In the inline <script> at the top of the JS section (before any other JS), add:
    ```js
    const SUPABASE_URL = 'PASTE_PROJECT_URL_HERE';
    const SUPABASE_ANON_KEY = 'PASTE_ANON_KEY_HERE';
    const supabaseClient = supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);
    ```

    Replace PASTE_PROJECT_URL_HERE and PASTE_ANON_KEY_HERE with the values from the checkpoint above.

    Note: The anon key is safe to include in frontend code — Supabase designed it to be public. Data is protected by Row Level Security policies (set up in Phase 2).
  </action>
  <verify>Open index.html in browser, open DevTools console → type `supabaseClient` → should return a Supabase client object, no errors.</verify>
  <done>AC-2 and AC-3 satisfied: SDK loaded, client initialised</done>
</task>
