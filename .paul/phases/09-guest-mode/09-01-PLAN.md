<objective>
## Goal
Add a "Continue as guest" option to the auth screen so riders can try the full calculator without creating an account. Guests get the same calculator experience but save functionality is replaced with sign-up prompts.

## Purpose
New riders landing on the app should be able to try it immediately without committing to an account. Reducing sign-up friction increases conversion — guests who see the calculator working are more likely to sign up to save their results.

## Output
- "Continue as guest" button on the auth overlay (below Google sign-in)
- `isGuest` global (sessionStorage-backed — clears when browser closes)
- `enterGuestMode()` function
- Guest restrictions: save buttons show "Sign in to save", Saved tab shows sign-up CTA, Settings tab shows sign-in prompt
- Guests can navigate all tabs and use all calculators — only saving is gated
</objective>

## Key locations in index.html

**Auth overlay** (~line 4932): `#auth-overlay` — contains `.auth-card` with logo, tabs (Log In / Sign Up), login form, signup form. Currently both forms have a Google sign-in button followed by nothing. The guest button goes below the last Google button in each form — or better, as a single link at the bottom of `.auth-card` outside both forms (always visible regardless of which tab is active).

**Auth state** (~line 4202): `supabaseClient.auth.onAuthStateChange` — `applyAuthState()` checks `session`. If no session → shows auth overlay. We need to intercept this: if `sessionStorage.getItem('rr_guest') === '1'`, skip the overlay and enter guest mode instead.

**`currentUser` global** (~line 3593): `let currentUser = null;` — add `let isGuest = false;` on the next line.

**`switchTab()`** (~line 4231): Currently calls `renderSaves()` only if `currentUser`. Guests hitting the Saved tab need a dedicated empty state.

**Save trigger buttons:**
- Suspension: `#save-trigger-btn` (~line 2431) — `onclick="openSaveNameRow()"`
- Tyre: `#tyre-save-trigger-btn` (~line 2580) — `onclick="openTyreSaveNameRow()"`

Both should show a toast or inline message for guests instead of opening the save flow.

**`renderSaves()`** (~line 3849): Returns early if `!currentUser`. Saved tab `#saved-list` will be empty for guests — needs a sign-up CTA rendered there.

**Settings tab**: Shows user email + avatar. For guests, show a sign-in prompt.

    function enterGuestMode() {
      isGuest = true;
      sessionStorage.setItem('rr_guest', '1');
      const overlay    = document.getElementById('auth-overlay');
      const appContent = document.getElementById('app-content');
      overlay.style.display = 'none';
      appContent.style.display = '';
      applyGuestRestrictions();
      if (typeof showOnboarding === 'function') showOnboarding();
    }

    function exitGuestMode() {
      isGuest = false;
      sessionStorage.removeItem('rr_guest');
    }

    function openAuthFromGuest(tab = 'signup') {
      exitGuestMode();
      const overlay    = document.getElementById('auth-overlay');
      const appContent = document.getElementById('app-content');
      appContent.style.display = 'none';
      overlay.style.display = 'flex';
      authShowTab(tab);
    }

    function applyGuestRestrictions() {
      // Save buttons — change to guest state (handled inline by onclick guards)
      // Saved tab — show CTA when tab opens (handled in switchTab)
      // Settings — handled in switchTab
    }
