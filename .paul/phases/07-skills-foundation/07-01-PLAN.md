## Key codebase facts (read index.html to verify before writing)

**Structure:** Single file — index.html contains all CSS (`<style>`), HTML, and JS (`<script>`). No build tools, no modules.

**Data:**
- `suspensionData` const (~line 2750) — JSON blob with `brands.{brand}.forks[]` and `brands.{brand}.shocks[]`
- Each model: `model`, `series`, `travel_mm[]`, `air_spring`, `sag_percent.{min,max}`, `max_pressure_psi`, `pressure_chart[]`, `formula_psi?`, `damping?`, `volume_spacers_supported`
- `pressure_chart` entries: `{ min_kg, max_kg, min_psi, max_psi }` — interpolate linearly
- `formula_psi: 'weight_lbs'` — overrides chart; result = round(weightKg × 2.2046)
- `damping` object: `{ damper, rebound?, rebound_lsr?, rebound_hsr?, lsc?, hsc?, compression?, ride_style_adjust? }`
- Each click adjuster: `{ starting_clicks: [{min_kg, max_kg, clicks}], total_clicks }`
- `ride_style_adjust`: `{ xc_trail: {rebound, lsc}, all_mountain: {...}, enduro_dh: {...} }` — delta applied to base clicks
- `tyreData` const (~line 2630) — tyre brands/models with pressure modifiers
- `SYMPTOM_FIXES` const — 6 symptom objects with fixes array and component field
- `BRAND_LABELS` — `{ rockshox, fox, dvo, ohlins, manitou }` → display names

**Key functions:**
- `calculate()` — main entry point, reads form → calls `buildResultHTML()` → populates result divs
- `buildResultHTML(brand, model, weightKg, type)` — builds full result card HTML including sag chip + ? button
- `computePSI(model, weightKg)` — returns PSI number (formula or interpolation)
- `interpolatePSI(chart, weightKg)` — linear interpolation within pressure_chart
- `buildDampingHTML(model, weightKg, type)` — builds damping section HTML
- `getClicksForWeight(startingClicks, weightKg)` — finds base clicks for weight
- `applyStyleDelta(clicks, total, delta)` — applies ride style adjustment
- `computeDampingSnapshot(model, weightKg)` — returns compact `{ damper, rebound, lsc, hsc, ... }` for saving
- `switchTab(tab)` — activates tab panel + nav btn, calls tab-specific render (renderSaves, renderFixitTab)
- `renderFixitTab()` — lazy, runs once (fixitRendered guard)
- `toggleSymptom(id)` — one-open accordion for Fix It cards
- `openSagVerify(btn)` / `closeSagVerify()` / `calcSagCorrection()` — sag modal
- `renderSaves()` / `renderTyreSaves()` — renders saved cards
- `pendingCalc` — global set after calculate: `{ weightKg, rideStyle, fork?: {brand, model, psi, damping}, shock?: {...} }`
- `currentUser` — Supabase user object or null
- `cachedSaves` — array of suspension saves from DB
- `currentRideStyle` — 'xc_trail' | 'all_mountain' | 'enduro_dh'

**Tab system:**
- Tabs: setup, saved, fixit, settings
- Each tab: `#tab-{name}` panel + `#nav-{name}` button
- `.tab-panel` / `.nav-btn` — active class toggled by switchTab()
- Setup sub-tabs: suspension / tyres (switchSetupSubTab)
- Saved sub-tabs: suspension / tyres (switchSavedSubTab)

**Modal pattern (established in Phase 5):**
- Static HTML shell with ID (#sag-modal-backdrop)
- `data-*` attrs on trigger button carry per-instance values
- `openX(btn)` reads dataset, populates modal, adds `.open` class
- `closeX()` removes `.open`, resets state

**Supabase:**
- Client init ~line 2627 with SUPABASE_URL + SUPABASE_ANON_KEY
- Saves table: `{ id, user_id, name, weight_kg, ride_style, fork_brand, fork_model, shock_brand, shock_model, created_at }`
- Auth: email/password; `currentUser` set in onAuthStateChange listener

**Theming:**
- CSS custom properties: `--text`, `--text-secondary`, `--text-muted`, `--surface`, `--surface2`, `--border`, `--orange`, `--accent`
- `.light` class on `<html>` for light mode (dark is default)

**Brands covered:** RockShox, Fox, DVO, Öhlins, Manitou

    # Suspension Domain Knowledge

    Load this skill when working with suspension data, PSI calculations, sag targets, or damping settings.

    ## Sag Targets (by ride style)

    | Component | XC/Trail | All Mountain | Enduro/DH |
    |-----------|----------|--------------|-----------|
    | Fork | 20–25% | 25–30% | 30–35% |
    | Shock | 25–30% | 28–32% | 30–35% |

    Note: Values vary by model — always use model.sag_percent.min/max from the data as the authoritative source.

    ## PSI Calculation Logic

    Two methods, in priority order:
    1. **formula_psi: 'weight_lbs'** — `round(weightKg × 2.2046)` — used by some RockShox models
    2. **pressure_chart interpolation** — `interpolatePSI(model.pressure_chart, weightKg)` — linear interpolation within the matching weight range

    If neither applies: model has no pressure data (rare).

    ## Damping Click Conventions

    - **0 clicks = fully open (fast)** — least damping
    - **Max clicks = fully closed (slow)** — most damping
    - Rebound: "add clicks" = slower rebound
    - Compression: "add clicks" = more compression resistance
    - Ride style adjustments are **deltas** applied to the base weight-matched value
