<objective>
## Goal
Make the post-calculate save CTA visually obvious and make the saved tab's empty state actionable — so riders notice they can save a result and know exactly what to do when they have no saves yet.

## Purpose
After a rider calculates, the save strip appears silently below the results — easy to miss on a quick scroll. And in the Saved tab, empty slot placeholders are static dashed cards with no interactive affordance. Both friction points reduce save conversions, which are the primary retention hook in RideReady.

## Output
- `@keyframes stripEnter` animation fires when save strip becomes visible (slide up + fade)
- Saved tab header shows live save count ("Saved · 1/3") via `renderSaves()` update
- Empty slot placeholders become `<button>` elements that navigate to the Setup tab, with a chevron and cleaner hint text
</objective>
