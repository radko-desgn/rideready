<objective>
## Goal
Resolve three high-visibility UX gaps: nav/content naming inconsistency, weak auth-screen brand/guest entry, and a dead-end guest experience in the Settings tab.

## Purpose
The app is feature-complete for v0.3 but has small surface-level inconsistencies that undermine trust and make key entry paths harder to find. Fixing these removes friction for new users (guest entry, sign-up moment on the auth screen) and resolves a confusing mismatch between the nav label and the tab it opens.

## Output
- index.html updated (CSS + HTML + no new JS)
- Bottom nav "Fix It" → "Feel" (aligns with "Suspension Feel" section header)
- Auth card: logo 48px → 80px; "Continue as guest" promoted from dim link to a visible third option
- Settings tab: guest banner converted to CSS class, gains a "Log In / Create account" CTA
</objective>
