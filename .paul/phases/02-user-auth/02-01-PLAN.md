<objective>
## Goal
Add full user authentication to RideReady — email/password sign up + log in, Google OAuth, sign out, and auth state management that gates the calculator behind login.

## Purpose
User accounts are the foundation for cloud-synced settings (Phase 3) and email collection for marketing. Every registered user's email lands in Supabase automatically.

## Output
- Auth modal (sign up / log in / Google) overlaid on the existing UI
- Supabase Auth fully wired (signUp, signInWithPassword, signInWithOAuth, signOut)
- Auth state listener (onAuthStateChange) showing/hiding UI based on login state
- Google OAuth configured in Supabase dashboard
</objective>


<task type="auto">
  <name>Task 2: Wire Supabase Auth logic</name>
  <files>index.html</files>
  <action>
    In the inline JS (after the existing supabaseClient initialisation), add:

    1. **onAuthStateChange listener** — runs on page load:
       - If session exists: hide #auth-overlay, show calculator, show logout button
       - If no session: show #auth-overlay, hide calculator content, hide logout button

    2. **Sign Up handler** (form submit):
       ```js
       const { error } = await supabaseClient.auth.signUp({ email, password });
       if (error) show error message;
       else show "Check your email to confirm your account";
       ```

    3. **Log In handler** (form submit):
       ```js
       const { error } = await supabaseClient.auth.signInWithPassword({ email, password });
       if (error) show error message (e.g. "Invalid email or password");
       // onAuthStateChange fires automatically on success
       ```

    4. **Google OAuth handler** (button click):
       ```js
       await supabaseClient.auth.signInWithOAuth({
         provider: 'google',
         options: { redirectTo: 'https://rideready-two.vercel.app' }
       });
       ```

    5. **Sign Out handler** (logout button click):
       ```js
       await supabaseClient.auth.signOut();
       // onAuthStateChange fires automatically
       ```

    Do NOT modify any existing calculator JS functions.
    Avoid: calling getSession() on every page interaction — use onAuthStateChange only.
  </action>
  <verify>Open browser console on localhost:8080 — no JS errors. Auth state listener fires on load.</verify>
  <done>AC-1, AC-2, AC-4, AC-5 logic satisfied</done>
</task>
